CLAUDE.md Guide

The CLAUDE.md file is the primary orchestration document for the Claude Agent SDK. It serves as a persistent memory layer and rulebook for how the agent should behave within a specific repository.

info Executive Summary

Think of CLAUDE.md as the "System Prompt" for your codebase. It defines the technical stack, coding conventions, and common CLI commands the agent can utilize during its lifecycle.

Core Structure

Every benchmark-ready repository should contain a .claude/CLAUDE.md file with the following sections:

example_claude_md.yml 
# Project Rules & Context

## Technical Stack
- Next.js 14 (App Router)
- Tailwind CSS & Shadcn UI
- Prisma ORM with PostgreSQL

## Coding Standards
- Use Functional Components with TypeScript
- Follow the "Clean Architecture" pattern in src/lib/
- Ensure all components are accessible (ARIA labels)

## Common Commands
- Build: npm run build
- Test: npm test
- Bench: claude-bench exec --all

Configuring the SDK

To enable specialized benchmarking, you can inject metadata that the Claude Test Bench uses to categorize runs. Add this block to your CLAUDE.md to enable automatic evaluator mapping.

rule Strict Mode

Forces the agent to fail if any linting error occurs during the benchmark cycle.

memory Context Locking

Prevents the agent from modifying specified configuration directories.

Writing Custom Rules

Harnessing the power of instruction density.

v1.2 Stable
01

Define Clear Boundaries

Clearly state which files the agent is authorized to edit and which are "Read-Only".

02

State Patterns Explicitly

Instead of "Write good tests," use "All tests must use Vitest and include 100% branch coverage."

03

Provide CLI Context

List exactly how to run the project. This prevents the agent from hallucinating script names.
